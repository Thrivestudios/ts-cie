Demo of recent data visualization work.
demo02
Two-column search: query tags on left, result artists on the right. Wires indicate how often the artist is assigned each tag. Artists are assigned a score based on the rank for each tag and sorted descending.
demo03
Three-column search: query artists on left, query tags center, result artists on the right. Artists on the left populate the query tags with each of their top three tags. All tags are then used to find and rank artists as in demo02.
demo04
Four-column search: query artists on left, query tags mid-left, result artists mid-right, and cluster tags (result tags) on the right. First two columns work as in demo03. The top tags for each result artist are loaded and the resulting bi-partite graph is clustered using a Markov Chain. Clusters with more than one artist member have their tags added to the fourth column. The green lines indicate cluster membership.
demo05
Switched to using a home-grown algorithm to cluster the result artists. The algorithm ranks result tags by maximizing the number of related artists and minimizing the number of other tags those artists are related to. This results in much better clustering and allows us to drop result tags which don't help the user narrow their search.
